Brand Concept
Site Name

APIぺディア

APIPedia

APIの百科事典。日本語で「調べる、理解する、使いこなす」を一つの場所で実現する。
「API」+「Encyclopedia（百科事典）」の造語で、包括性と信頼感を表現。

「探す」を「使う」に変える、日本語APIカタログ
Vision

日本のすべての開発者が、言語の壁なく世界中のAPIを発見・理解・活用できる世界をつくる。APIの民主化を通じて、個人開発者やスタートアップのプロダクト開発を加速し、日本のソフトウェアエコシステム全体の生産性を底上げする。

Tone & Voice

実用的で親しみやすい技術ガイド。
難解な専門用語を避け、具体的なコード例と平易な日本語で解説する。上から教えるのではなく、隣で一緒に調べる同僚のようなスタンス。正確性と網羅性を重視しつつ、初学者にも門戸を開く。

Lean Canvas
1 Problem -- 課題
  • 発見困難: 日本語でAPIを横断検索できる場所がなく、用途に合うAPIを見つけるのに膨大な時間がかかる
  • 理解障壁: 海外APIのドキュメントは英語中心。仕様・料金体系・制限事項の理解にストレスと誤解が生じる
  • 比較不能: 同カテゴリのAPI同士を統一フォーマットで比較する手段がなく、技術選定の判断根拠が属人的になる
既存の代替手段
  • RapidAPI（英語のみ）
  • Google検索 + 個人ブログ
  • GitHub awesome-apis（未整理）
2 Solution -- ソリューション
  • カテゴリ別カタログ: 決済・認証・AI・地図など用途別に整理された日本語APIデータベース
  • 統一フォーマット解説: 概要、料金、制限、コード例を共通テンプレートで日本語提供
  • 比較表・選定ガイド: 同カテゴリAPIの機能・料金・制限を一覧比較できるマトリクス
3 Unique Value Proposition
日本語で、探せる。わかる。選べる。

世界中のAPIを日本語で網羅的に検索・理解・比較できる唯一のカタログサイト。統一フォーマットによる解説と実践的なコード例で、APIの「発見」から「実装」までを最短距離でつなぐ。

差別化の3軸
  • 完全日本語対応
  • 統一比較フォーマット
  • 実装直結のコード例
4 Unfair Advantage -- 圧倒的な優位性
  • 先行者優位: 日本語APIカタログ市場で先行しSEO資産とブランドを構築
  • コミュニティ効果: ユーザーレビュー・実装事例の蓄積はネットワーク効果を生む
  • 構造化データ: 統一フォーマットで蓄積されたAPI情報は機械的な比較・推薦に活用可能
5 Customer Segments -- 顧客セグメント
  • 個人開発者: 副業・趣味でWebアプリ/モバイルアプリを開発する層（最大ボリューム）
  • スタートアップ: MVP開発でAPI活用を検討する少人数チーム
  • 駆け出しエンジニア: API連携を学びたいプログラミング学習者
  • 企業の技術選定担当: API導入の比較検討を行う中小企業のリード開発者
アーリーアダプター
  • 技術ブログを書く個人開発者
  • API連携のチュートリアルを求める学習者
6 Key Metrics -- 主要指標
  • MAU: 月間アクティブユーザー数
  • API掲載数: カタログ収録API件数
  • 検索→詳細遷移率: 検索からAPI詳細ページへの転換率
  • 滞在時間: 詳細ページ平均滞在時間
  • リピート率: 月内再訪問率
  • SEO流入数: オーガニック検索からの月間流入
7 Channels -- チャネル
  • SEO: 「○○ API 日本語」等のロングテールキーワードで検索上位獲得
  • 技術コミュニティ: Zenn / Qiita / note での解説記事公開
  • SNS: X（旧Twitter）での新規API紹介・Tips発信
  • 開発者イベント: ハッカソン連携・LT登壇
  • ニュースレター: 週刊API新着・アップデート情報配信
8 Cost Structure -- コスト構造
  • コンテンツ制作: API調査・日本語ドキュメント作成・コード例制作の人的コスト
  • インフラ: ホスティング、CDN、ドメイン（静的サイトで最小化）
  • SEO / マーケティング: コンテンツマーケティング、SNS運用
  • ツール: 分析ツール、エディタ、CI/CD
9 Revenue Streams -- 収益の流れ
  • アフィリエイト: API提供元への送客による成果報酬（有料APIの登録誘導）
  • スポンサード掲載: API提供企業による掲載枠・優先表示
  • プレミアムコンテンツ: 詳細比較レポート・選定ガイドの有料提供
  • 広告: 開発者向けターゲティング広告（Carbon Ads等）
Strategic Notes
Phase 1 -- MVP（0-3ヶ月）

目標: 50件のAPIを掲載し、月間1,000 UU達成

静的サイトとして構築。決済・認証・AI・地図・メッセージングの5カテゴリから着手。各APIは統一テンプレートで「概要 / 料金 / 制限 / コード例 / 類似API」を掲載。SEO最適化を徹底し、ロングテールキーワードで検索流入を獲得する。

Phase 2 -- Growth（3-6ヶ月）

目標: 200件のAPIを掲載し、月間10,000 UU達成

カテゴリ比較表の導入、検索機能の強化、ニュースレター配信開始。Zenn/Qiitaでの技術記事をフックにコミュニティ形成を促進。APIプロバイダーへのスポンサード掲載の営業を開始する。

Phase 3 -- Scale（6-12ヶ月）

目標: 500件以上のAPIを掲載し、月間50,000 UU達成

ユーザーレビュー機能、API推薦エンジン、実装チュートリアルの拡充。コミュニティ貢献によるコンテンツ拡張の仕組みを構築。収益化モデルの本格稼働。

Key Risks & Mitigations
  • RISK コンテンツ制作の継続負荷 → テンプレート化 + AI補助で効率化
  • RISK API情報の陳腐化 → 定期クロール + コミュニティ報告の仕組み
  • RISK 大手プレイヤーの日本語対応 → 日本市場特化の深いコンテンツで差別化
  • RISK 収益化までの資金 → 静的サイト運用で固定費を最小化
